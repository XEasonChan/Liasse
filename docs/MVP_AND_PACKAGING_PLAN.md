# MVP 与 Electron 打包规划

目标用户不是开发者，所以最终产品应当是：下载、双击、选择录音、等待、拿到逐字稿。用户不需要自己安装 Ollama、Python、ffmpeg、模型框架或命令行工具。

## 技术路线判断

当前最适合 Apple Silicon 的核心转录路线是 `mlx-qwen3-asr`：

- 直接在 MLX/Metal 上运行 Qwen3-ASR，适合 M1/M2/M3/M4。
- 核心 ASR 不依赖 PyTorch。
- 默认 0.6B 适合 MVP 快速验证，1.7B 用于质量模式。
- 支持长音频、时间戳、批量文件、多种导出格式。
- 可作为 Python sidecar，也可通过内置 HTTP server 被 Electron 调用。

官方 `qwen-asr` PyTorch 路线仍保留为备用，但它更偏 CUDA/vLLM，不是 Mac 小白产品的首选路径。

## 第一阶段：转录能力验证 MVP

目标：先证明“用户给 3-4 小时访谈录音，本地能稳定产出可用逐字稿”。

范围：

- 只做本地 ASR，不做摘要。
- 默认模型：`Qwen/Qwen3-ASR-0.6B`，用于速度和安装包体积验证。
- 高质量模式：`Qwen/Qwen3-ASR-1.7B`，用于抽样对比。
- 输入：wav 优先；m4a/mp3 需要打包 ffmpeg 或在 App 内转码。
- 输出：Markdown、JSON、SRT。
- 批处理：支持多文件队列，单个文件失败不影响后续文件。

验证集：

- 5 分钟短样本：快速回归。
- 30 分钟样本：观察进度、时间戳、导出。
- 3-4 小时真实访谈：验证长音频稳定性。
- 至少包含普通话、夹杂英文术语、低音量或远距离录音。

验收指标：

- App 不联网也能处理已下载模型。
- 处理过程有明确进度、当前文件、剩余队列。
- 输出文件能直接用于人工校对。
- 长音频失败时保留错误信息，不吞掉已完成文件。
- 0.6B 与 1.7B 在同一段样本上的质量差异可被人工评估。

## 第二阶段：基本用户体验

首屏就是工作台，不做营销页。

核心交互：

- 拖拽或选择多个录音文件。
- 显示队列：文件名、时长、状态、进度、预计剩余时间。
- 模式选择只暴露给普通用户能理解的词：
  - 快速转录
  - 高质量转录
  - 带发言人识别
- 输出位置可改，默认在用户文档目录下创建项目文件夹。
- 完成后展示结果预览，提供“打开文件夹”和“继续校对”。

需要避免：

- 让用户看到 Python、pip、Ollama、模型路径这些概念。
- 首次打开就要求填 token。
- 把技术错误原样甩给用户。

## 第三阶段：发言人识别

发言人识别是独立能力，不属于 ASR 模型本体。

MVP 处理：

- 初版可先关闭，避免 pyannote 权重授权和体积问题阻塞转录 MVP。
- 第二个可测版本接入 `mlx-qwen3-asr` 的 `--diarize`/pyannote 路线。
- 用户界面只显示“尝试区分发言人”，并允许用户事后把 `SPEAKER_00` 改名为“采访者”等。

产品风险：

- pyannote 默认模型可能需要 Hugging Face 授权，不能成为小白用户的第一步。
- 更适合做成“专业功能包”：首次使用时由 App 解释并完成一次性下载。

## 第四阶段：本地摘要但不要求 Ollama

不要求用户安装 Ollama。可选路线：

- 内置 `llama.cpp`/`llama-server` 二进制和 GGUF 量化模型。
- 或内置 `mlx-lm` 与 MLX 量化模型，保持 Apple Silicon 路线统一。

建议顺序：

1. 转录 MVP 先不打开摘要。
2. 加一个无需模型的“访谈概览”：时长、说话人片段、关键词候选、长段落索引。
3. 再加入真正 LLM 摘要 sidecar，模型随 App 下载或随离线安装包提供。

摘要模型建议：

- 默认：小体积 4B 量化模型，用于普通用户。
- 专业模式：8B/14B 量化模型，用于学术访谈摘要。

## Electron 生产架构

推荐架构：

- Electron 主进程：管理窗口、文件权限、任务生命周期。
- Renderer：React/TypeScript 工作台。
- Python/MLX sidecar：负责 ASR、diarization、导出。
- 通信：本地 HTTP 或 stdio JSON-RPC。
- 模型目录：放在用户 Application Support 下，支持首次下载和离线导入。

打包策略：

- MVP 内测包：Electron + Python sidecar + 0.6B 模型下载器。
- 离线包：Electron + Python sidecar + 0.6B 模型权重 + ffmpeg。
- 专业包：额外提供 1.7B ASR、diarization、摘要模型。

为什么不把所有东西一开始塞进安装包：

- 1.7B ASR、发言人识别、摘要模型叠加后体积会很大。
- 普通用户首次体验更需要“先成功转录一个文件”。
- 模型包可以做成 App 内管理，既满足离线伦理要求，也不牺牲下载体验。

## 当前代码状态

本仓库已有：

- Tkinter 桌面 MVP，可双击 `.command` 启动。
- 多文件批处理。
- `mlx-qwen3-asr` 后端适配。
- 官方 `qwen-asr` 和 `faster-whisper` 备用后端。
- Markdown、JSON、SRT 导出。
- 演示任务和轻量测试。

下一步真实能力验证需要：

1. 安装 Python 3.10+。
2. 安装 `requirements-mlx.txt`。
3. 下载 0.6B 或 1.7B 模型权重。
4. 用 5 分钟样本跑通端到端转录。
5. 再上 30 分钟和 3-4 小时样本。
